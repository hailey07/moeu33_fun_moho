原帖 (2009年7月14日)



好了，我想我终于准备好发布我的“微调键”工具集了。除了使其兼容 6.0 版本的新功能外，我还添加了一些额外的选项，并对所有功能进行了优化，使其响应速度更快。特别是，以前需要运行长达一分钟才能完成的菜单脚本（用于为一组图层创建合并时间线），现在几乎是瞬间完成的。

顺便说一下，这些工具应该适用于 5.6 和 6.0 版本，但 5.6 版本会覆盖插值模式，并且图层排序键不会进行调整。

您可以从这里下载完整的工具和脚本集……

rt_nudge_keys_v0.1.zip

该工具集包含以下按钮工具：

rt_nudge_keys_left、

rt_nudge_keys_right、

rt_global_nudge_keys_left <

rt_global_nudge_keys_right >

rt_prev_keyframe -

rt_next_keyframe +

以下菜单脚本：

rt_create_consolidated_timeline_layer.lua

以及以下包含一些常用函数的实用工具脚本：

rt_utilities.lua

我还包含一个名为 _tool_list_fragment.txt 的文件，其中显示了我推荐的按键分配。

我承认这些图标质量不太好，所以如果您愿意帮忙制作更好的图标，请尽管去做。

希望它足够简单易懂，但这里还是简要说明一下它的使用方法。

1. 在任何图层中，按下“,”键，光标所在位置或右侧的所有按键都会向左移动一位。按下“.”键，按键会向右移动一位。如果当前图层有子图层，则其下方的所有图层也会受到影响。如果光标左侧紧邻一个按键，则不会执行向左移动的操作。

2. 在任何图层中，按住Shift键并按下“,”或“.”键，文档中的所有按键（包括相机按键）都会移动一位。文档的结束帧也会自动调整。

3. 在任何图层中，运行rt_create_consolidated_timeline_layer脚本，父图层（如果存在）和同级图层（如果存在）中的所有按键都会合并到一个特殊的非渲染图层中，该图层与原始图层并排显示。每当您在此图层中向左或向右移动按键时，其父图层及其子图层都会自动受到影响。这个脚本也可以在顶层运行，对监控全局微调非常有用。

4. rt_nudge_keys_left.lua 文件底部有一些选项变量，我计划将来用一个单独的选项脚本来实现它们。我现在太累了，没法一一解释，不过其中一个比较有趣的是“enable_fake_curvature_channel”。问题在于，脚本界面目前无法获取每个点的曲率通道，但我找到了一种相当稳健的方法来模拟它：假设每个曲率关键帧都有一个对应的点位置关键帧。这种方法有点取巧的地方在于，它使用一个高帧的单曲率关键帧动作来屏蔽当前的曲率通道。这就是为什么它默认是禁用的。
